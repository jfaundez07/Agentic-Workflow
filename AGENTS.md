# Agentic Workflow

OpenCode agent definitions and architecture blueprint system (The Architect). **Config/doc repo** — no application code, no build scripts, no tests, no CI, no linter.

## Agent Files

- **`agents/*.md`** — each file has YAML frontmatter (mode, permissions, temperature) that OpenCode reads as executable config. Be careful editing.
- **No `opencode.json`** in this repo. Config lives entirely in markdown frontmatter.
- **Two primary agents** (run standalone): `tech-lead.md` (orchestrator) and `the-architect.md` (blueprint designer, does NOT write code).
- **Six sub-agents** (dispatched via `task` tool): analyst, commiter, developer, planner, reviewer, tester.
- **Permission model**: Tech Lead has `bash: true` (full shell access), `write`/`edit` restricted to `docs/plans/` and `docs/workflow-log.md`. Reviewer is the only sub-agent with `bash: false`.

## Tech Lead Pipeline

Sub-agents run sequentially: Planner → Developer → Tester → Reviewer. Planner creates `docs/plans/plan-<id>.md` (sequential IDs from 1); subsequent agents read it. Analyst runs independently to produce `docs/analysis.md`. Commiter is a fallback, not a pipeline step.

The Tech Lead logs every handoff to `docs/workflow-log.md` and passes structured JSON context between sub-agents. Always dispatch sub-agents via the `task` tool.

## The Architect System

4-phase workflow: Discovery → Deep Dive → Architecture → Generate. Resources under `resources/the-architect/`:
- **6 archetypes**: saas-webapp, marketing-site, mobile-app, api-backend, internal-tool, content-platform
- **8 building blocks**: auth, db, deployment, api design, frontend, testing, styling, state
- **3 questionnaires** (phase-1, phase-2, phase-3), **2 templates** (blueprint, AGENTS.md)

## Runtime Artifacts

- **`.opencode/` is gitignored** (root `.gitignore`). It holds runtime artifacts generated by agents (analysis, plans, etc.).
- **Installation**: `cp -r agents/ ~/.config/opencode/agents && cp -r resources/ ~/.config/opencode/resources`

## Path Resolution

The Architect's knowledge files use `~/.config/opencode/resources/the-architect/...` paths. These resolve from the user's home config at runtime, not the repo. Do not rewrite to local paths.

## Commit Convention

Conventional Commits with scoping. Prefixes: `feat:`, `fix:`, `refactor:`, `chore:`, `docs:`. Always scope to the changed component (e.g., `fix(tester):`, `refactor(agents):`).
